# Domain completeness checklist

Use this to ensure each domain flow is fully specified.

## Software feature
- Requirements + functional + technical + architecture
- Test plan + quality contract
- Decision log + progress log
- Diagrams (context, container, component)

## Bug fix
- Repro steps and severity
- Technical spec + test plan
- Regression tests

## PR review
- Comment audit + response plan
- Test validation
- PR report

## Data science
- Metrics + data sources
- Bias/fairness checks
- Monitoring plan

## Design
- Deliverables + approval cadence
- Accessibility checks
- Asset formats

## Humanities
- Thesis + scope
- Source reliability checks
- Structured outline

## Business
- Assumptions + sensitivity
- Scenario analysis

## Legal
- Jurisdiction + compliance
- Audit and retention rules

## Learn
- Research plan and outputs
- Bias check and confidence level
