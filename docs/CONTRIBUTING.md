# Contributing

## Branching
- Use Gitflow branches: `feature/*`, `fix/*`, `chore/*`.
- Squash-merge into `develop`.

## Commits
- Use Conventional Commits (e.g. `docs: add style guide`).

## Docs
- Update `docs/INDEX.md` when adding new docs.
- Place reports under `docs/reports/`.

## Tests
- Run `npm test` before pushing.
